(북로그) 자바 객체 지향의 원리와 이해

February 2, 2019

2 minute read


스프링에 입문하려는 (JAVA OOP)개발자에게 추천해주고 싶은 책이다.

 당장 스프링 프로젝트를 구현하는 것이 목적인 분들에게는 어쩌면 이 책은 도움이 되지 않을 수 있다. 그와 관련된 내용은 책의 후반부에나 나오기 때문이다. 대신 이 책은 독자에게 JAVA OOP 개발에 대한 전체적인 시선을 알려주고 스프링을 그 맥락에서 설명한다.

 자바를 처음 공부했던 내가 자바에 대해 느낀점은 "어렵다"였다. 의무적으로 인터페이스를 만들고 구현 클래스를 만들고 생성자를 호출했지만 도대체 왜 그렇게 해야하는 지는 이해할 수 없어서 어려웠다.(물론 전과생이어서 적응상의 문제일 수 있다.) 이 책은 객체지향 프로그래밍이라는 큰 관점에서 해당 키워드들을 설명해 준다. 나는 이 책을 통해 abstract, ,interface, this, package등의 키워드들을 객체지향이라는 큰 관점에서 이해함으로서 자바에서 느꼈던 어려움을 완전히 날려버릴 수 있었다. 또한 해당 키워드들을 더욱 올바르게 사용할 수 있게 되었다.

 나아가 책은 일관성 있게 객체지향적 관점에서 객체지향 설계원칙(SOLID), 디자인 패턴, 스프링 프레임워크에 대해서 설명한다. 다만 책의 물리적 한계로 개념의 핵심과 맛보기 정도의 설명만 다룬다. 저자는 그 한계를 인식하고 해당 주제에 대한 책들을 추천하거나 블로그 자료들을 추가로 링크한다. 내가 이 책을 좋아하는 이유는 이 부분인데 그렇게 함으로서 이 책은 JAVA OOP 개발의 일종의 지도가 되어준다는 점이다. 내가 무엇을 모르는 지를 아는 것 또한 상당히 중요하기 때문이다. 또한 곳곳에 선배 개발자로서의 경험이 묻어있는 설명이 좋았다. "SRP(단일 책임 원칙)을 지키지 않으면 if문을 남발하게 됩니다."와 같은 설명은 설계 원칙의 중요성을 더 피부로 느낄 수 있었다.

막상 작성하고 보니 북리뷰가 아닌 추천사와 같은 글이 되버렸다.
혹시나 나와 같이 스프링 입문자라면 이 책을 통해 스프링을 OOP적 관점에서 이해하였으면 좋겠다.
