XML 네임스페이스

February 16, 2019     ""

2 minute read

네임스페이스의 개념과 사용

XML에서 태그를 자유롭게 정의할 수 있다는 점은 장점이지만, 중복 태그가 생길 수 있는 문제가 있다. 중복 태그로 빋어질 수 있는 이름 충돌을 해결하기 위해 등장한 개념이 네임스페이스이다. 네임 스페이스는 각 요소명과 속성명을 정확히 인식할 수 있는 개념을 제공한다.

네임스페이스 선언

1.기본 네임스페이스 선언 어떤 요소에 대하여 xmlns 속성을 사용해서 네임스페이스를 선언한 경우, 그 요소 내용 중 네임스페이스 접두사를 붙이지 않고 기술한 하위child 요소에도 자동으로 상위parent 요소에서 선언한 네임스페이스를 적용한다.(만일, 모든 요소가 단 하나의 네임스페이스를 공통적으로 따른다면 기본 네임스페이스로 선언해서 사용하는 것이 효과적이다.

기본 형식

기본 네임스페이스를 다중으로 사용한 예

<?xml version="1.0" encoding="utf-8"?>
<students xmlns="http://www.koreaxml.ac.kr/2015/students">
    <student>
        <sid> 100 </sid>
        <name> 홍 현 </name>
        <age> 30 </age>
        <profile>
            <html xmlns="http://www.w3.org/1999/xhtml">
                <head>
                    <title> 나의 소개 </title>
                </head>
                <body> 활발한 사회활동 중 </body>
            </html>
        </profile>
    </student>
</students>

2.기본 네임스페이스 취소 선언 이미 선언한 기본 네임스페이스를 취소할 때는 대상이 되는 요소에 대해 xmlns 속성값에 URI_Reference 대신 빈 문자열(” “)을 지정하여 기본 네임스페이스를 선언하면 된다.(하위 요소 모두 적용)

ex)

<?xml version="1.0" encoding="utf-8"?>
<students xmlns="http://www.koreaxml.ac.kr/2015/xml">
    <student>
        <sid> 100 </sid>
        <name> 홍 현 </name>
        <age> 30 </age>
        <address xmlns=" ">서울 면목동</address>
    </student>
</students>

3.접두사를 사용한 네임스페이스 선언 xmlns: 속성은 네임스페이스 접두사를 사용해서 특정 요소에 대해 네임스페이스를 선언하는 경우이다. 네임스페이스 접두사는 XML 문서 중에서 사용하는 요소명이나 속성명과 xmlns: 속성값으로 지정하는 URI 참조를 관련시키기 위해 사용한다.

기본 형식 <(prefix_name:) element_name xmlns:prefix_name = “URI_Reference”>

ex) 선언 예

(잘 확인 <st:students xmlns:st="http://www.koreaxml.ac.kr/2016/students" )

<?xml version="1.0" encoding="utf-8"?>
<st:students xmlns:st="http://www.koreaxml.ac.kr/2016/students" xmlns:pf="http://www.w3.org/1999/xhtml">
    <st:student>
        <st:sid> 100 </st:sid>
        <st:name> 홍 현 </st:name>
        <st:age> 30 </st:age>
        <st:profile>
            <pf:html>
                <pf:head>
                <pf:title> 나의 소개 </pf:title>
                </pf:head>
                    <pf:body> 활발한 사회활동 중 </pf:body>
            </pf:html>
        </st:profile>
    </st:student>
</st:students>

한정한 이름(qualified name, qname)

XML 네이슴페이스에서는 요소명이나 속성명 앞에 네임스페이스 접두사와 콜론(:)을 붙여서 각각의 요소나 속성이 어떤 네임스페이스에 속하는 것인가를 정의하게 된다. 이와 같이 네임스페이스 접두사를 붙인 요소나 속성명을 한정한 이름(qualified name)이라고 한다.

ex) <prefix_name:local_name>
