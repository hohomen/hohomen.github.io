스프링 DI(Dependency Injection)

April 20, 2019     ""

3 minute read

스프링의 가장 기본적인 기능은 객체를 생성하고 초기화하여 필요로 하는 곳에 제공하는 것인데, 이 중심에는 DI(Dependency Injection)라는 의존에 대한 설계 패턴이 적용되어 있다.

목차
1. DI와 스프링 (가장 기본적 의존 생성)
2. 스프링 DI 설정  
  2.1 XML을 이용한 DI 설정
    keyword : <bean> <property> <constructor-arg>
  2.2 자바 코드를 이용한 DI 설정
    keyword : @Configuration과 @Bean @ImportResource
  2.3 XML 설정과 자바 코드 설정 함께 사용하기
    keyword : <context:annotation-config/> , @ImportResource
3. 애노테이션을 이용한 객체 간 의존 자동 연결
    keyword : @Autowired , @Resource , @Inject
4. 컴포턴트 스캔을 이용한 빈 자동 등록
    keyword : @Component , @Service , @Controller , @Repository
5. 스프링 컨테이너 추가 설명.


1. DI와 스프링
타입에 의존한다는 것은 다른 말로 해당 타입의 객체를 사용한다는 것을 뜻한다.
의존 객체를 직접 생성하는 방식과 달리 DI(Dependency Injection)는 의존 객체를 외부로부터 전달받는 구현 방식이다.

스프링은 객체를 생성하고 각 객체를 연결해주는 조립기 역할을 하게 된다.
public static void main(String[] args) {
    String configLocation = "classpath:applicationContext.xml";
    AbstractApplicationContext ctx = new GenericXmlApplicationContext(configLocation);
    Project project = ctx.getBean("sampleProject", Project.class);
    project.build();
    ctx.close();
}


GenericXmlApplicationContext 클래스는 이 XML 파일에 정의된 설정 정보를 읽어와 객체를 생성하고 각각의 객체를 연결한 뒤에 내부적으로 보관한다. 이렇게 생성한 객체를 보관하기 때문에 스트링을 객체 컨테이너(Object Container)라고도 부른다.



XML을 이용한 스프링 설정을 보면 컨테이너가 생성할 객체를 지정하기 위해 <bean> 태그를 사용하는데, 이 이름에서 알 수 있듯이 스프링 컨테이너가 생성해서 보관하는 객체를 스프링 빈(Spring Bean) 객체라고 부른다. 물론, 스프링 빈 객체라고 해서 특별한 객체는 아니고 일반적인 자바 객체와 동일하다.



2.1 XML을 이용한 DI 설정
<bean> 태그는 스프링 컨테이너가 생성할 객체에 대한 정보를 지정할 때 사용된다. 태그의 주요 속성은 id와 class다.
class 속성은 스프링 컨테이너가 생성할 객체의 클래스의 이름을 값으로 갖는다.
id 속성의 값은 스프링 빈 객체의 고유한 이름으로 사용한다.



<bean> 자식 태그
<property> 태그는 프로퍼티 설정 방식을 사용하는 경우에 사용한다.
name 속성을 이용해서 프로퍼티 이름을 지정한다. <property> 태그를 사용하면, set프로퍼티이름( ) 형식의 메서드를 이용해서 값을 설정한다. ex) setDataSource( )
프로퍼티의 값은 1. value 속성 혹은 태그<value> 그리고 2.ref 속성 혹은 태그<ref>을 이용해서 값을 설정할 수 있다.
ref 속성은 다른 빈 객체를 사용하는 경우에 사용한다.



<bean> & 자식 태그 예제) context-root.xml 코드 중 일부

<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource" />
    <property name="configLocation" value="classpath:/mybatis-config.xml"></property>
    <property name="mapperLocations" value="classpath:mappers/*Mapper.xml"></property>
</bean>
<bean id="dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
    <property name="driverClassName" value="net.sf.log4jdbc.sql.jdbcapi.DriverSpy"></property>
    <property name="url" value="jdbc:log4jdbc:mysql://127.0.0.1:3306/cultura"></property>
    <property name="username" value="root"></property>
    <property name="password" value="choi1201"></property>
    <property name="connectionProperties">
        <props>
            <prop key="useUnicode">yes</prop>
            <prop key="characterEncoding">utf8</prop>
        </props>
    </property>
</bean>

객체를 생성할 때 생성자에 전달할 파라미터 값을 설정해주어야 할 경우 사용한다. 예제)

new User("dhchoi", "1234");
<bean id="user2" class="com.cultura.domain.User">
    <constructor-arg> <value>dhchoi</value></constructor-arg>
    <constructor-arg value="1234"></constructor-arg>
</bean>

(스프링은 XML 외에 자바코드와 그루비 코드를 이용해서 설정하는 방법을 제공하고 있다.)

2.2 자바 코드를 이용한 DI 설정

@Configuration과 @Bean을 이용한 빈 객체 생성 자바 코드를 이용한 설정은 XML을 이용한 설정보다 더 직관적이다. 스프링이 자바 코드를 스프링 설정 정보로 사용하도록 하려면 다음과 같이 @Configuration과 @Bean 애노테이션을 사용하면 된다.

@Configuration
public class Config {

    @Bean
    public User user1() {//메서드 이름이 식별자이다.
        return new User("dhchoi", "1234");
    }
    
    @Bean(name = "user2")//식별자를 메서드 이름이 아닌 user2로 사용
    public User user() {
        return new User("madvirus", "qwer");
    }
}

@Configuration 설정 클래스를 이용해서 스프링 컨테이너를 생성할 때에는 o.s.context.anotation.AnnotationConfigApplicationContext 클래스를 사용한다.(XML에서 GenericXmlApplicationContext클래스와 같은 역할)
예)

AnnotationConfigApplicationContext ctx = new AnnotationConfigApplicationContext(Config.class);
User user1 = ctx.getBean("user1", User.class);
System.out.println(user1.getId();

XML 코드와 자바 코드의 비교.

@Bean
    public UserRepository userRepository() {
        UserRepository userRepo = new UserRepository();
        userRepo.setUsers(Arrays.asList(user1(), user()));
        return userRepo;
    }

<bean id="userRepository" class="net.madvirus.spring4.chap02.UserRepository">
    <property name ="users">
        <list>
            <ref bean="user1"/>
            <ref bean="user2"/>
        </list>
    </property>
<bean>
