REST 기본과 REST Controller

May 20, 2019     ""

3 minute read

이 포스트에서는 간단히 REST를 정리하고 REST Controller에 대해서 정리합니다.
프론트엔드(client)에서의 REST 처리는 AJAX(JQUERY) 정리 및 실전 코드 분석에 정리하였습니다.

REST에 대한 자세한 개념 정리는 그런 REST API로 괜찮은가에서 이해할 수 있습니다.




REST(Representational state transfer)



개념.
URI는 하나의 고유한 리소스(Resource)를 대표하도록 설계된다.
의미
URI는 반드시 그에 상응하는 데이터 자체. ex)/boards/123



REST API
-외부에서 위와 같은 방식으로 특정 URI를 통해서 사용자가 원하는 정보를 제공하는 방식.
-최근에 Open API에서 많이 사용되면서 REST 방식으로 제공되는 외부 연결 URI를
REST API라고 하고, REST 방식의 서비스 제공이 가능한 것을 ‘Restful’ 하다고 표현합니다.



어디에 사용하나?
결과 데이터만을 서버에서 제공하는 방식은 데이터를 이용하는 클라이언트 측에서의 기능이
많아지는 경우에 사용됩니다. 모바일 환경이나 HTML5, Ajax등을 이용하는 경우에 많이 사용됩니다.






컨트롤러 정리.
@RestController는 JSP와 같은 뷰를 만들어 내지 않는 대신에 데이터 자체를 반환한다.



단순 문자열
@RestController에서 문자열 데이터는 기본적으로 브라우저에는 ‘text/html’타입으로 처리됩니다.
ex)
    @RequestMapping(“/hello”)
    public String sayHello(){
         return “Hello Wrold”;
    }



객체를 JSON으로
@RestController에서 객체는 자동으로 JSON으로 처리될 수 있습니다.
(pom.xml에 jackson-databind(2.5.4) 라이브러리를 추가해 주어야 한다.)
(<- 객체를 JSON 타입으로 변환하거나 반대의 작업을 할 때 사용.)
(그렇지 않으면 스프링에서 객체 변환에 실패해서 406 Not Accepatable 에러)



ex)@RequestMapping(“/sendVO”)
    public SampleVO sendVO(){
        SampleVO vo = new SampleVO();
        vo.setFirstName(“길동”);
        vo.setLastName(“홍”);
        vo.setMno(123);

        return vo;
    }



Content-Type 확인
브라우져 개발자도구-네트워크의 Response Headers로 Content-Type 등 전송 정보를 확인할 수 있습니다.
+Response Headers
    Content-Type: application/json;charset=UTF-8
    Date: Sat, 25 Jul 2015 06:30:35 GMT
    Server: Apache-Coyote/1.1
    Transfer-Encoding: chunked



컬렉션 타입으로 반환하는 경우
List는 배열로 표현 [{ : , : , : }, { }, … { }]
Map은 key와 value로 구성 { “0” : { : , : , : }, “1” : {..},  .. }
{ : , : , : } = sampleVO타입의 객체.



Map 예제)

  @RequestMapping("/sendMap")
  public Map<Integer, SampleVO> sendMap() {

    Map<Integer, SampleVO> map = new HashMap<>();

    for (int i = 0; i < 10; i++) {
      SampleVO vo = new SampleVO();
      vo.setFirstName("길동");
      vo.setLastName("홍");
      vo.setMno(i);
      map.put(i, vo);
    }
    return map;
  }







Rest 방식의 Controller 작성
@PathVariable : URI의 경로에서 언하는 데이터를 추출하는 용도로 사용.
ResponseEntity 타입은 개발자가 직접 결과 데이터 + HTTP 상태 코드를 제어할 수 있는 클래스이다.
(안 쓰고 리턴한 뒤 예외처리 해주어도 괜찮을 것 같다. 이때는 @ResponseBody 사용)



사용 예)

@RequestMapping(value = "/all/{board_id}", method = RequestMethod.GET)
    public ResponseEntity<List<ReplyVO>> list(@PathVariable("board_id") Integer board_id) {

      ResponseEntity<List<ReplyVO>> entity = null;
      try {
        entity = new ResponseEntity<>(service.list(board_id), HttpStatus.OK);
      } catch (Exception e) {
        e.printStackTrace();
        entity = new ResponseEntity<>(HttpStatus.BAD_REQUEST);
      }
      return entity;
    }

@RequestBody : 전송된 JSON 데이터를 객체로 변환해 주는 애노테이션.
((커맨드 객체)@ModelAttribute와 유사한 역할을 하지만 JSON에서 사용된다는 점이 차이)

사용 예)

@RequestMapping(value = "/replies", method = RequestMethod.POST)
    public String register(@RequestBody ReplyVO vo) {
        service.create(vo);       
        return "redirect:/sboard/list";
    }

@ResponseBody : 자바 객체를 JSON 형식이나 XML 형식의 문자열로 변환할 때 사용한다.
(리턴 타입이 HTTP의 응답 메세지로 전송) 사용 예)

@RequestMapping("/board/read")
public @ResponseBody ProductVO doJSON(){    
    ProductVO vo = new ProductVO("샘플상품",30000);     
    return vo;      
}
