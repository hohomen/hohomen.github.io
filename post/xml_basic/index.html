XML 기본

February 18, 2019     ""

2 minute read

  • 마크업(mark up)이란 문서 내용 이외에 문서의 구조를 정의하거나 표현하기 위한 부가적인 정보이다.
  • 다양한 플랫폼과 애플리케이션이 공존하는 인터넷에서 문서의 호환성은 매우 중요한 요소이다.(ex)oracle과 mysql의 호환)
  • XML은 SGML로부터는 정보의 구조적 기술과 정보 내용과 표시 정보를 분리한 구조를 이어받고, HTML로부터는 인터넷의 이용 환경을 이어받았다.

XML 일반 기술

1. 네임스페이스
서로 다른 XML 문서 간에 사용하는 태그를 구벼할 수 있게 하는 기술이다. 여러 XML 문서를 통합하여 새로운 하나의 XML 문서를 생성하는 겨웅 '이름 충돌' 문제를 해결하기 위한 것이다.

2. DTD(Document Type Definition)와 XML 스키마
DTD와 XML 스키마는 XML 문서의 구조를 정의하는 기술이다. DTD는 기존의 XML 문법과 다르고 제한적인 문서 구조만 정의할 수 있는 단점이 있다. 이러한 문제점을 해결하기 위해 제안된 것이 XML 스키마이다.

3. CSS, XSL, XSLT
CSS와 XSL은 XML 문서를 화면에 표시하기 위한 기술이다. CSS는 브라우저가 XML문서의 데이터를 어떠한 형태로 출력할 것인지 인식하게 한다. XSL은 XSLT를 사용하여 XML문서를 브라우저가 인식할 수 잇는 다른 형태의 문서로 변환하여 나타낸다. 예를 들면, XML문서를 HTML이나 WML과 같은 문서로 변환하는 것이다.

4. XPath
XML 문서의 특정 위치에 대한 주소 지정 방법을 지원하는 기술이다. XML 문서로부터 데이터를 질의하기 위한 구조로 사용하며, XSLT, XLink, XPointer등에서 활용한다.

5. XLink
XLink는 XML 문서 간의 연결 방법을 정의한 기술로 HTML에서 제공하는 하이퍼링크보다 많은 기능을 제공한다. HTML에서는 <a> 태그를 통해서만 현재 문서에서 다른 문서로 이동하거나 현재 문서 내에서 'name' 속성이 정해진 태그로 이동할 수 있다. 그러나 XLink는 태그 사이를 이동할 수 있는 다양한 방식을 제공한다. 또한 HTML은 단방향 링크만 제공하는데 비해, XLink는 단방향은 물론 양방향 리크도 제공한다. 따라서 문서의 링크 관계를 좀더 정확히 파악할 수 있고 링크 자체의 의미를 나타낼 수도 있다.

6. XPointer
