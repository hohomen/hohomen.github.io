함수(JavaScript Basic)

February 19, 2019     ""

2 minute read

함수는 코드의 집합을 나타내는 자료형입니다.

1.익명함수: 이름이 없으므로 변수에 넣어 사용해야 합니다.(point= 변수)
var 함수 = function() { };

자바스크립트에서 함수는 한의 자료형이지만 여타, 자료형과 다르게 뒤에 괄호를 열고 닫음으로써 코드를 실행합니다. = '함수를 호출'

함수(); // 익명함수를 호출할 때.


2.선언적 함수: 이름이 있는 함수.
function 함수( ) {

}

#선언적 함수와 익명 함수의 차이점.
익명함수는 변수로 선언했기 때문에 오류 발생.
반대로  선언적 함수를 사용하면 코드가 정상적으로 실행.

    cf. function 함수( ) { alert('함수 A'); }

4.매개변수
*자바스크립트는 함수를 생성할 때 함수에서 지정한 매개변수 개수보다 많거나 적은 매개변수를 사용하는 것을 허용합니다.
(매개변수보다 많은 수를 선언하면 일반적으로 추가된 매개변수는 무시합니다.)
(원래 함수에서 선언할 수 있는 매개변수 숫자보다 적게 선언하면 지정하지 않는 매게변수는 undefinded로 입력됩니다. ex)prompt())


일부 함수는 이렇게 지정한 매개변수보다 많거나 적게 매개변수를 사용하는 점을 이용하기도 합니다.
5.가변 인자 함수.
매개변수를 선언된 형태와 다르게 사용했을 때, 매개변수를 모두활용하는 함수를 뜻합니다.


cf. 가변 인자를 사용한 함수.
function sumAll() {
            var output = 0;
            for (var i = 0; i < arguments.length; i++) {
                output += arguments[i];
            }
            return output;
        }
        alert(sumAll(1, 2, 3, 4, 5, 6, 7, 8, 9));
}//결과 : 45

6.리턴 값.
return 키우드를 사용하면 값을 지정하지 않아도 함수를 호출한 곳으로 돌아갑니다.
function returnFunction() {
            return;
            alert('문장 A');
            alert('문장 B');
        }
        // 함수를 호출합니다.
        var output = returnFunction();
        alert(typeof (output) + ' : ' + output);
}// 결과 undefined : undefined

7.내부 함수
내부 함수는 이러한 충돌을 막는 방법입니다.(<- 프로그램의 규모가 커지면 커질수록 다른 사람과 함께 프로그램을 개발하며 그 와중에 여러 가지 충돌이 발생합니다.)

8.콜백 함수
자바스크립트에서는 함수도 하나의 자료형이므로 매개변수로 전달할 수 있습니다.
function callTenTimes(callback) {
            for (var i = 0; i < 10; i++) {
                callback();
            }
        }
        var callback = function () {
            alert('함수 호출');
        };
        callTenTimes(callback);
} // 경고 화면을 10번 출력

함수를 매개변수로 전달하는 것은 분명한 이유가 있습니다. 이번 절에서 보았듯이 특정 함수를 여러번 실행할 수 있고, 9장에서 이벤트를 배우면 "이렇게 사용하는 것이구나"라고 이해할 수 있습니다.

익명함수. (콜백 함수는 익명 함수로 사용하는 경우가 많습니다.)
function callTenTimes(callback) {
            for (var i = 0; i < 10; i++) {
                callback();
            }
        }
        callTenTimes(function () {
            alert('함수 호출');
        });

9. 함수를 리턴하는 함수

function returnFunction() {
            return function () {
                alert('Hello Function .. !');
            };
        }
        // 함수를 호출합니다.
        returnFunction()();

함수를 리턴하는 함수를 사용하는 가장 큰 이유는 클로저(closure) 때문입니다.
10. 클로저
지역 변수를 남겨두는 현상.
function test(name) {
            var output = 'Hello ' + name + ' .. !';
            return function () {
                alert(output);
            };
        }
        // 출력합니다.
        test('JavaScript')();

클로저와 관련된 것은 직접 여러 가지 프로그램을 접하면서 "이러한 경우에 사용해야 한다"라는 것을 느껴야 합니다.
이책에서는 7장 캡슐화 부분에서 다시 한 번 클로저와 관련된 내용을 살펴봅니다.
