ajax(jQuery) 정리 및 실전 코드 분석

April 20, 2019     ""

2 minute read


Intro
$.ajax( ) 메서드는 내부에서 XMLHttpRequest 객체를 만들어 Ajax를 수행합니다.(직접 XMLHttpRequest 객체를 만들 때보다 간단합니다.)

Syntax.문법(두 가지 형태로 사용할 수 있다.)
1. $.ajax(url, options); // 일반적 형태
$.ajax('/data.html',{
    success: function (data) {
        $('body').append(data);
    }
});

2. $.ajax(options);
//매개변수 url은 옵션 속성으로 지정할 수도 있다.(옵션은 객체로 이루어져있다.)
$.ajax({
    url: '/data.html',
    success: function (data) {
        $('body').append(data);
    }
});

메서드 옵션(객체의 속성은 대부분 자동으로 지정한다.)
주요 속성은 다음과 같다.
async : 동기 비동기를 지정합니다.
complete(xhr, status) : Ajax 완료 이벤트 리스너를 지정합니다.
data : 요청 매개변수를 지정합니다.
error(xhr, status, error) : Ajax 실패 이벤트 리스너를 지정합니다.
success(data, status, xhr) : Ajax 성공 이벤트 리스너를 지정합니다.
timeout : 만료 시간을 지정합니다.
type : Get 또는 Post 등을 지정합니다.
url : 대상 URL을 지정합니다.
(속성에 대한 자세한 정보는 http://api.jquery.com/jquery.ajax/)


실전 코드를 갖고 좀 더 메서드 옵션을 실용(심화)적으로 정리해 봅시다.

        $.ajax({
              type:'put',
              url:'/replies/'+replyId,
              headers: { 
                    "Content-Type": "application/json",
                    "X-HTTP-Method-Override": "PUT" },
              data:JSON.stringify({replytext:replytext}), 
              dataType:'text', 
              success:function(result){
                  console.log("result: " + result);
                  if(result == 'SUCCESS'){                    
                      alert("수정 되었습니다.");
                  }
              }    
        });

"X-HTTP-Method-Override": "PUT"
많은 브라우저가 GET, POST 방식만을 지원하기 때문에 REST 방식을 제대로 사용하려면 브라우저가 지원하지 않는 상황을 고려해야 합니다. 그 해결책으로 브라우저에서 POST 방식으로 전송하고, 추가적인 정보를 이용해서 PUT, PATCH, DELETE와 같은 정보를 같이 전송하는 것입니다. 이를 'Overloaded POST'라고 합니다.
1. Ajax를 이용해서 전송하는 경우 "X-HTTP-Method-Override": "PUT"를 headers 정보에 포함시켜줍니다.
2. <form> 태그를 이용해서 데이터를 전송하는 경우 POST 방식으로 전송하되 '_method'라는 추가적인 정보를 이용합니다. 
    <input type ="hidden" name="_method" value="PUT">
    
cf. 스프링에서는 이런 처리를 위해서 설정할 수 있는 HiddenHttpMethodFilter를 제공하는데, 웹 애플리케이션(web.xml) 내에서 Filter로 설정해서 처리합니다.
    <filter>
        <filter-name>hiddenHttpMethodFilter</filter-name>
        <filter-class>org.springframework.web.filter.HiddenHttpMethodFilter</filter-class>
    </filter>
    <filter-mapping>
        <filter-name>hiddenHttpMethodFilter</filter-name>
        <url-pattern>/</url-pattern>
    </filter-mapping>
data:JSON.stringify({replytext:replytext})
자바스크립트 객체를 JSON 문자열로 변환합니다.(웹 서버에 데이터를 전송할 때는 문자열이어야 합니다.)

dataType:'text', 
dayaType 속성은 입력하지 않아도 된다. dataType 속성을 입력하지 않으면 적절한 형태로 데이터가 변환된다. 이 속성을 넣고 안 넣고의 차이는 다음과 같다.

속성을 넣었을 때 Text 출력
[
    {
        "name" : "우유",
        "price" : "2000"
    }
    
속성을 넣지 않았을 때 Text 출력
[object Object], [object Object], [object Object], [object Object]


추가적인 jQuery Ajax 메서드
$.getJSON()  :  get 방식으로 Ajax를 수행해 JSON 데이터를 가져옵니다.
syntax : $.getJSON(url,data,success(data,status,xhr))
$.get( )  :  
syntax : $.get(URL,data,function(data,status,xhr),dataType)
$.post( )  :  post방식으로 Ajax를 수행합니다.
syntax : $.post(URL,data,function(data,status,xhr),dataType)

REST 방식에서 $.post( )를 사용하지 않는 이유
$.post의 형식은 $.post(URL,data,function(data,status,xhr),dataType) 이렇습니다. 즉 이 메소드는 일반적인 데이터 전송(<form> 처리된 데이터 전송) 방식에 적합합니다. 이 메소드는 headers: { "Content-Type": "application/json"}과 같은 옵션을 지정할수 없습니다. REST 방식에서 사용한다면 415(지원되지 않는 타입) 상태 코드가 발생합니다.
