HTML에서 JavaScript Load

February 20, 2019     ""

2 minute read


본 글은 생활코딩의 내용을 정리 요약한 것이다.
먼저 HTML에서 JavaScript Load하는 방법은 3가지로 나눌 수 있다.

1. 인라인
alert('hello world') 자바스크립트 코드
onclick=" ", onchange 등등 HTML 속성
장점 : 태그 안에 분명히 있다는 것.
단점 : html, 즉 정보를 나타내는 코드에 제어 코드가 들어가있다.
유지보수에 어렵다. 짬뽕. js가 필요할 때 js에 찾아가고, html필요할 때 html에 찾아가는게 좋다.

2. script태그
<script>까지는 html태그. 이제 script 코드가 나온다는 신호
장점 : 인라인보다 유지보수보다 용이해짐. (정보와 제어를 분리)
단점 : 그럼에도 정보를 담당하는 문서안에 제어를 담당하는 코드가 들어있다.

3. 외부파일로 분리.
<script src="./script.js"></script> (-> html코드이다.)
장점: 정보를 담당하는 파일과 제어를 담당하는 파일을 완전히 분리
만약 여러 HTML파일이 JS 파일을 공유한다면 유지보수에 용이하다.(중복의 제거)
HTML과 js의 결합도를 낮춤.
js파일을 별도로 빼놓게 되면 브라우져에 의해서 캐쉬처리되는 경우가 많다. latency를 줄일 수 있다.
서버, 클라이언트의 통신량을 줄이고 html문서를 경량화할 수 있다.



4. 스크립트 파일은 어디에 위치시키는 것이 좋은가?
보통 head 태그(웹 브라우저 화면에는 보이지 않지만 웹 브라우저가 알아야 할 정보들을 입력한다.)
혹은 body 태그 가장 아래에 위치시킨다.

A. script를 head 태그에 위치시킬 수도 있다. 하지만 이 경우는 오류가 발생한다.

<!DOCTYPE html>
<html>
<head>
    <script src="script2.js"></script>
------------아래와 위는 정확히 같은 의미이다.-------------
    <script>
    var hw = document.getElementById('hw');
    hw.addEventListener('click', function(){
    alert('Hello world');
    </script>
오류가 발생하는 이유는 브라우져는 아직 body 태그를 인식하지 않았기 때문에
id=hw는 null로 인식된다.
</head>
<body>
    <input type="button" id="hw" value="Hello world" />
</body>
</html>

B. 따라서 아래와 같이 script2.js의 코드를 수정해야 한다.
window.onload = function(){
    var hw = document.getElementById('hw');
    hw.addEventListener('click', function(){
        alert('Hello world');
    })
}
이것의 단점은 브라우저는 스크립트에 대한 로딩이 끝나고 이것을 해석하기 전까지 나머지 코드를 실행하지 않는다. 
---> 렌더링이 지연된다.(사용자는 하얀 화면만 보인다.)
---> 반대로 script태그가 아래에 있다면 브라우져가 해석한 코드들이 전부 출력된 다음에 스크립트가 실행되기 때문에 사용자는 빠르게 사용가능

C. <script>를 아래에서 로드할 때는 window.onload가 필요가 없다.
    var hw = document.getElementById('hw');
    hw.addEventListener('click', function(){
        alert('Hello world');
    })
