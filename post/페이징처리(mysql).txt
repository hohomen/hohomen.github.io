<a> 태그를 이용하는 경우 최대 장점은 검색엔진에 노출이 쉬워진다는 점입니다. 링크가 처리되면 한 번에 모든 연결정보를 파악할 수 있습니다.

<form> 처리를 보다 간결하게 할 수 있도록 링크에는 최소한의 정보를 이용 빠르게 개발할 수 있다.

페이징 처리 원칙
- 반드시 Get 방식만 사용
- 조회 화면에서 '목록 가기' 구현 
(history.go(-1)과 같은 방식으로 처리하는 경우도 있지만, 게시물 리스트 자체가 프레임이나 <iframe>의 내부에 들어 있는 경우에는 사용할 수 없다.)
- 필요한 페이지 번호만 제공

필요한 기술적 내용
- 페이징 처리를 위한 SQL
- 데이터 개수 파악을 위한 SQL 
- 자바스크립트 혹은 <a> 태그를 통한 이벤트 처리


(페이징 작업은 자바로 작성되는 코드의 양도 중요하지만)
#더 중요한 것은 jsp와 같은 뷰에서의 처리입니다.

계산을 매번 JSP 등에서 처리할 수도 있겠지만, 좀 더 편하게 사용하기 위해서는 별도의 클래스를 설계해서 처리하는 것이 좋습니다.(대부분의 회사에서 이를 처리하는 클래스를 이미 작성해 둔 경우가 많습니다. PageMaker)
