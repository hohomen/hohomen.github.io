DTD 문서 제작 기본

January 30, 2019     ""

2 minute read

Well-formed Document(정형식 문서)와 Valid Document(유효한) XML 문서


|정형식 문서|유효한 문서| 정형식 문서|

Well-formed Document(정형식 문서)는 흔히 말하는 XML 문서에 속한다.
XML 문법과 규칙에 맞게 만들어진 오류가 없는 문서이다.

Valid Document(유효한 문서)
XML 문서를 표준화할 목적으로, 문서 형태 정의(DTD)에 따라 만든 XML문서.

DTD(Document Type Definition)는 문서를 형태를 정의하는 문서를 말한다.
XML 문서의 구조를 명시적으로 선언하며, XML 문서가 잘 만들어진 유효한 문서인지 확인하는 역할을 한다.
1. 작성한 문서를 이해하는 데 도움.(문서 가독성)
2. 문서가 제대로 만들어졌는 지 확인할 수 있다.
3. 문서를 교환할 때 인식성이 높아진다.

XML 문서에 DTD를 적용하여 사용하는 방법은 크게 세 가지로 구분할 수 있다.
(내부DTD, 외부DTD, 혼합DTD)

1.내부DTD

별도의 DTD 문서를 가지지 않으므로 사용하기 편리하지만, 다른 XML 문서에 적용하기에는 좋지 않은 방법이다.

<!DOCTYPE root_element[
    마크업 선언(요소 선언 | 속성선언 | 개체선언 | 노테이션 선언 | PI | 주석)
]>

사용 예)

<?xml version="1.0"?>
<!DOCTYPE db[
    <!ELEMENT db (person*)>
    <!ELEMENT person (name,age,email)>
    <!ELEMENT name (#PCDATA)>
    <!ELEMENT age (#PCDATA)>
    <!ELEMENT email (#PCDATA)>
]>

<db>
    <person>
        <name>Alan</name>
        <age>42</age>
        <email>agb@gamil.com</email>
    </person>
    <person>
        <name>Ptan</name>
        <age>24</age>
        <email>ptn@gmail.com</email>
    </person>
</db>

2.외부 DTD

외부 DTD 문서를 별도로 작성하여 XML 문서에 연결하는 방법이다. 별도의 DTD 문서를 관리해야 하지만, 여러 XML 문서에 공통적으로 적용하기에 적합한 방법이다.

외부 DTD 선언

<!DOCTYPE root_element source location1 location2 

>

source : SYSTEM 혹은 PUBLIC이라는 키워드 사용 가능 (SYSTEM의 경우엔 특정 조직에서 별도의 용도로 사용, PUBLIC으로 선언한 엔티티는 공식적인 공용 식별자를 필요로 한다.)

location1 혹은 location2 : 외부에 선언된 DTD 파일의 위치

사용예) myBatis mapper.xml 문서의 일부이다.

<!DOCTYPE mapper
  PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
  "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
PUBLIC을 명시한 곳(-//mybatis.org//DTD Mapper 3.0//EN)에서 DTD를 이용할 수 없는 경우에는 SYSTEM 위치에 있는 DTD(http://mybatis.org/dtd/mybatis-3-mapper.dtd")를 사용한다.

3.혼합 DTD 선언

<!DOCTYPE root_element source location1 location2[ 
    내부 DTD
    마크업 선언(요소 선언 | 속성선언 | 개체선언 | 노테이션 선언 | PI | 주석)
]>xml.ac.kr/2015/xml">
    <student>
        <sid> 100 </sid>
        <name> 홍 현 </name>
        <age> 30 </age>
        <address xmlns=" ">서울 면목동</address>
    </student>
</students>
