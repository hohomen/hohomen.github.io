(ant) build.xml 작성법

January 29, 2019     ""

3 minute read

자바를 이용해 자료구조를 공부하면서 빌드 도구인  ant를 사용할 일이 생겼다.
여기에  ant를 사용한다면 필수적으로 알아야할 build.xml 문서 작성에 대한 정리를 하고자 한다.

빌드 시작 명령어인 

>ant

가 실행되면  ant는 기본적으로 현재 작업 디렉터리에서 build.xml이라는 빌드 파일을 찾아 실행한다. (완성된 build.xml문서는 페이지 아래에 정리해 두었다.) 이제 그 내부 작성을 단계적으로 따라가보자.

<?xml version="1.0"> (xml 문서 선언)

1.프로젝트 정의하기

<project name="dms" default="compile" basedir=".">

name: 프로젝트 이름 default: 명령줄에서 매게 변수 없이 ant를 입력할 때 엔트가 빌드프로세스에서 컴파일 단계를 실행해야 한다는 뜻이다. basedir: 속성값을 ‘.‘으로 해주면, 이 빌드 파일에서 쓰는 다른 모든 경로는 build.xml이 들어 있는 디렉터리를 기준으로 한 상대 경로를 사용한다는 뜻이 된다.

2.재료 선택하기

프로퍼티(property)를 정의하여 엔트에 프로젝트 디렉터리 구조를 알려준다.

<property name="build.dir" location="build"/>
<property name="build.prod.dir" location="${build.dir}/prod"/>
<property name="build.test.dir" location="${build.dir}/test"/>
<property name="doc.dir" location="doc"/>
<property name="index.dir" location="index"/>
<property name="src.dir" location="src"/>
<property name="test.dir" location="test"/>
<property name="vendor.lib.dir" location="vendor/lib"/>

추가설명, 각각의 요소는 location 속성 값에 쓴 디렉터리와 name 값을 연결한다. location 속성이 가리키는 디렉터리들은 요소에서 정의한 basedir 속성 값을 기준으로 한 상대 경로로 표현한 디렉터리다.

${propertyName.dir}는 디렉토리를 상대적으로 선언하는 것이다. 이렇게 선언하는 것의 장점은 나중에 디렉터리의 위치가 바뀔 경우 propertyName.dir의 location의 위치만 바꿔주면 된다는 점이다.(ex. 여기서는 build –> bin)

3.재료 버무리기

(완전한 빌드를 생성하려면 모든 빌드 재료들을 지정하기 쉽게 만드는 것이 좋다.) 먼저 클래스 경로부터 시작하자.

<path id="project.classpath">
    <pathelement location="${build.prod.dir}" />
    <pathelement location="${build.test.dir}" />
    <fileset dir="${vendor.lib.dir}">
        <include name="*.jar"/>
    </fileset>
</path>

해설 요소는 project.classpath라는 경로를 생성한다. 이 경로는 나중에 빌드 프로세스의 컴파일 단계에서 사용할 것이다. 요소에서 빌드 디렉터리 두 개를 경로에 추가해서 자바 컴파일러가 모든 .class 파일을 찾아낼 수 있게 했다. 그 다음에 요소에서 모든 벤더 라이브러리 .jar 파일을 클래스 경로에 추가해서 .class파일이 사용할 벤더클래스들을 찾을 수 있도록 했다.

4.상 차리기

<target name="prepare">
    <mkdir dir="${build.prod.dir}" />
    <mkdir dir="${build.test.dir}" />
</target>

target은 지정한 순서에 따라 실행될 일련의 ‘task’들을 정의하는 빌드 단계이다. 은 명령줄에서 mkdir를 사용하는 것과 동일하게 작동하지만, 어떤 운영체제에서나 작동하고 만들 디렉터리가 이미 있더라도 상관없으며, 필요하다면 상위 디렉터리까지 만들어 준다는 점이 다르다.

5.본격적으로 시작하기.

<target name="compile" depends ="prepare">
    <javac srcdir="$src.dir}" destdir="${build.prod.dir}">
        <classpath refid="project.classpath" />
    </javac>
</target>

target의 실행 순서는 매우 중요하다. 왜냐하면, 컴파일 결과를 저장할 디렉터리 없이 소스를 컴파일할 수는 없기 때문이다. depends 속성으로 타깃 간 의존성을 만들어서 순서를 지정해야 한다. 이 속성에는 타깃 이름을 콤파(,)로 구분해서 나열한다. 예를 들어 위 코드에서 compile 타깃은 depends 속성에 prepare 타깃에 대한 의존성을 선언했다. 이것은 compile 타깃을 실행하면 소스 파일을 컴파일하기 전에 필요한 디렉터리를 만들기 위해 먼저 prepare 타깃을 실행한다는 뜻이다.

refid(=reference ID)

완성 문장.

<?xml version="1.0">
<project name="dms" default="compile" basedir=".">
    <property name="build.dir" location="build"/>
    <property name="build.prod.dir" location="${build.dir}/prod"/>
    <property name="build.test.dir" location="${build.dir}/test"/>
    <property name="doc.dir" location="doc"/>
    <property name="index.dir" location="index"/>
    <property name="src.dir" location="src"/>
    <property name="test.dir" location="test"/>
    <property name="vendor.lib.dir" location="vendor/lib"/>
        <path id="project.classpath">
            <pathelement location="${build.prod.dir}" />
            <pathelement location="${build.test.dir}" />
            <fileset dir="${vendor.lib.dir}">
                <include name="*.jar"/>
            </fileset>
        </path>
        <target name="prepare">
            <mkdir dir="${build.prod.dir}" />
            <mkdir dir="${build.test.dir}" />
        </target>
        <target name="compile" depends ="prepare">
            <javac srcdir="$src.dir}" destdir="${build.prod.dir}">
                <classpath refid="project.classpath" />
            </javac>
        </target>
</project>
