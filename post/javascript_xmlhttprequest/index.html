ajax (XMLHttpRequest)

March 20, 2019     ""

2 minute read


본 내용은, 도서 : 모던 웹을 위한 JavaScript jQuery 입문 3판의 내용을 정리한 것입니다.

1. XMLHttpRequest 객체
는 빈 편지지와 같습니다. 이 객체에는 수취인과 배송방식, 내용물을 넣을 수 있습니다.

a. open( ) 메서드로 편지지의 전송 위치와 방식을 지정합니다.
request.open(<전송 방식>, <경로>, <비동기 사용 여부 true 비동기, false 동기>)
request.open('GET', '/data.html', false);

b. send( ) 메서드로 편지를 보냅니다.
request.send( );

현재 /data.html은 HTML 형식의 문자열이죠? 따라서 innerHTML 속성에 넣어주면 응답받은 데이터를 곧바로 출력할 수 있습니다.
cf. url
/data.html 데이터를 HTML 형식으로 제공합니다.
/data.json 데이터를 JSON 형식으로 제공합니다.
/data.xml 데이터를 XML 형식으로 제공합니다.

c.responseText 편지가 성공적으로 보내져 답장이 왔다면 XMLHttpRequest 객체의 responseText 속성을 살펴봅니다.
document.body.innerHTML += request.responseText;
(결과, xml 코드는 책의 내용을 확인)
우유
2000
홍차
5000
커피
5000

2.동기/비동기 방식
동기방식: 데이터를 서버와 클라이언트가 같은 속도로 연계하여 동작하는 방식을 의미. send( ) 메서드에서 3ms 정도 소요되는 것을 확인할 수 있다.
비동기 방식: 데이터를 받는 동안에도 코드를 지속적으로 실행. send( ) 메서드에서 0ms~1ms 소요되는 것을 확인할 수 있다.

3 종합 코드 및 분석
        var request = new XMLHttpRequest();
        request.open('GET', '/data.html', true);
        request.send();
        request.onreadystatechange = function (event) { 
    //XMLHttpRequest의 상태가 변경될 때마다 이벤트를 호출.
            if (request.readyState == 4) {
                if (request.status == 200) {
                    document.body.innerHTML += request.responseText;
                };
            };
        };

readyState 속성 설명
0 : 처음으로 xmlhttp가 생성된 상태를 말합니다.(=request 객체 생성)
1 : 열기. open 메소드가 호출되었으나 send 메소드는 호출되지 않았음.
2 : 송신완료. send 메소드가 호출되었음. 요청이 시작되었음.
3 : 수신 중, 서버가 응답을 보내는 중임.
4 : 수신완료. 서버가 응답을 다 보내고 작업을 완료했음

HTTP Status Code
1XX - 처리중
200 - 서버가 정상으로 존재하고, 문제없이 응답을 받았을 경우,
3XX - 리다이렉트
400 - Bad Request
403 - 접근을 거부당한 경우
404 - 파일이 없을경우(예) xxxx.xml 파일이 존재하지 않을경우 )
500 - 서버에 오류가 발생했을 경우
